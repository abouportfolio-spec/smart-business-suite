# Contribution à Smart Business Suite 🚀

Merci de l'intérêt que vous portez à ce projet ! En tant qu'aspirant architecte logiciel, je tiens à ce que le code reste propre, modulaire et facile à maintenir.

## 🏗️ Principes de développement
Pour contribuer, vous devez respecter les principes suivants :

1. **Séparation des préoccupations** : 
   - Le code lié à la configuration globale va dans `core/`.
   - La logique métier (ventes, produits, stocks) va exclusivement dans `business/`.
   - Le design (CSS/Tailwind) est géré dans le module `theme/`.

2. **Qualité du code** :
   - Commentez vos fonctions complexes.
   - Respectez les conventions PEP 8 pour le code Python.

## 🚀 Comment contribuer ?
1. **Forkez** le projet.
2. Créez une branche pour votre fonctionnalité (`git checkout -b feature/ma-fonctionnalite`).
3. Faites vos changements et testez-les localement.
4. Effectuez un **commit** avec un message clair (ex: `feat: add export to PDF for sales`).
5. Ouvrez une **Pull Request**.
